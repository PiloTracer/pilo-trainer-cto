# Delivery & Systems

**Slug:** `delivery-systems`  
**Duration:** 6–8 weeks  
**Level:** Any CTO owning delivery outcomes

## Outcomes

- Roadmap that can survive a 40% cut
- Explicit architecture tradeoff memos
- Reliability ownership model (SLO mindset)
- Tech debt policy with economic framing

## Modules

### M1 — Roadmap integrity
- Drill: `roadmap-cut`
- Sources: Inspired (Cagan) for product pairing; internal STRATEGY docs if present

### M2 — Architecture tradeoffs
- Objectives: Options, irreversible choices, migration costs
- Drill: Write ADR-style memo for one live bet
- Sources: Documenting Software Architectures (Clements et al.) — selective; public ADR guides

### M3 — Reliability
- Drill: `incident` — SEV timeline + external/internal comms + postmortem quality
- Sources: Site Reliability Engineering (Google SRE book — free online); incident.io / public postmortem collections (cite URLs)

### M4 — Tech debt
- Drill: `tech-debt` — quantify cost of delay vs rewrite fantasy
- Exit: Cut roadmap + one ADR + one incident drill score ≥ pass

## Assessment

Numbers and tradeoffs required; no “we need a rewrite” without economics.
