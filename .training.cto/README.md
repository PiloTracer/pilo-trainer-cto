# `.training.cto/` — learner memory

All CTO training, consultancy, and progress artifacts for this project live here.

**Never** put learner-specific content into the framework tree (`skills/`, `curricula/`, `standards/`). Copy catalog curricula into `programs/` instead.

> **Note on this self-hosted instance:** this repo is the CTO Professor OS framework source. The history in `context/HANDOFF.md` through 2026-07-17 documents *framework-build work* (building the OS, releasing v0.0.1, merging safety rules) — not a real learner's training. `context/PROFILE.md` is intentionally left unfilled pending an explicit decision to use this repo as a real training project (see `.cursorrules`'s learner-pipeline intent guard). Don't read this history as genuine learner memory.
